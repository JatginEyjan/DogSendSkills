## 2. DS14 设计讨论 - 氧气与背包系统

### 氧气机制再审视

**问题发现**：
- 原始目的：解决"事件密度不够，玩家机械点击无聊"
- 环境崩溃方案的问题：可能造成死局（门远+格子少+氧气耗尽）

**Eyjan 的关键提问**：
- "氧气这个设定本身是为了解决什么问题呢？"

**重新定位**：
- 氧气应该是"软限制"而非"死线"
- 推荐方案：效率递减（氧气耗尽=减速+掉血，但还能撑）

### 背包系统设计

**现状问题**：
- `inventory: []` 空置，雷达/镐子只是全局状态
- 雷达无限用，无决策点
- 玩家可以"盲点"通关

**设计方案**：
```
村庄（配置）→ 地牢（锁定）→ 村庄（补给）

背包内容：
- 雷达（带电量，扫描消耗）
- 镐子（带耐久，挖掘消耗）
- 消耗品（血瓶、理智药、氧气罐）
- 特殊道具（炸弹、传送卷轴）

策略点：
- 进地牢前：带什么？带多少？
- 地牢中：用不用？什么时候用？
- 危急时：撤退还是赌一把？
```

**MVP 阶段**：
- Phase 1：基础背包（2格）+ 雷达电量
- Phase 2：镐子耐久 + 消耗品
- Phase 3：特殊道具
